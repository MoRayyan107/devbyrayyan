Project Summary

This project aims to develop a car dealership management system that allows both customers and employees to interact with the database. Customers can view available cars, search for specific models, and enter their details if they wish to purchase a car. Employees, on the other hand, can perform various tasks such as inserting, editing, grouping, and deleting records in the database.


The code has the follows:


  1. Imports and Setup:
    • 'Imports':

      Statements bring in necessary Python modules.

    • 'sqlite3':

      Module is used to interact with SQLite databases.

    • 'tabulate':

      It is imported to format database query results into tables.

    • 'pyttsx3':

      It is imported to enable text-to-speech functionality for spoken feedback.

  2. Function Definitions:
    • Table_Values():

      Inserts values into database tables. It interacts with the user to gather input for inserting records into tables like "available_cars", "car_models", and "car_features".

    • Edit_Values():

      Edits existing values in database tables. It allows users to modify records in the database based on their inputs.

    • Group():

      Groups records in tables based on specific criteria entered by the user. It retrieves records from tables like "car_models", "car_features", and "customer_details" and displays them in a grouped format.

    • Delete_Values():

      Deletes values from tables based on user input. It interacts with the user to identify the records to be deleted and removes them from tables like "available_cars", "car_models", and "car_features".

    • timedate():

      Displays a greeting message based on the current time. It determines whether it's morning, afternoon, or evening and greets the user accordingly.

  3. Main Program Flow:
    • Greeting message:

      The program starts by greeting the user with a message based on the current time.

    • User interaction:
      • Customers:

        Presented with options to view available cars, search for specific cars, or provide their details for purchasing a car.

      • Employees:

        Prompted to enter a passcode for access. Upon successful authentication, they can perform various operations such as inserting, editing, grouping, or deleting values from tables.

    • Database operations:

      The program interacts with the SQLite database to retrieve, insert, edit, group, or delete records based on user input.

    • User feedback:

      Feedback messages are provided to confirm actions or inform users of errors. Text-to-speech synthesis is used to speak messages aloud for enhanced user experience.

    • ASCII art:

      Visual elements like logos and banners are displayed using ASCII art to enhance the program's interface and aesthetics.

  4. Error Handling:

    • Basic error handling is implemented for password validation and user input to ensure smoother program execution and user interaction. For example, incorrect passwords prompt users to retry, and invalid inputs are handled to prevent crashes.
